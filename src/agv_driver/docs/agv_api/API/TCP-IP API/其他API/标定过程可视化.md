**请求**

- 编号: 6115(0x17E3)
- 名称: robot\_other\_calib\_allinone2\_req
- 描述: 标定过程可视化
- JSON 数据区: 根据插件定义

```**字段名**|**类型**|**描述**|**可缺省**``` :- | :- | :- |
||string|json字符串，具体根据标定插件的定义|否|
|文件二进制数据|文件二进制数据|文件二进制数据|是|


```需要注意，此 API 数据区分为两个部分： **json数据区以及文件二进制数据区** 。两种数据需要紧密排列。排列顺序为：json数据+文件二进制区。在此情况下，包头中的 **数据区长度** 为json数据长度+文件二进制数据长度。包头中的 **内部使用区域** 中的第2位和第3位需要设置为json数据区的长度。```


```包头详情可见 [此处 ](..\概述\API报文结构.md)，其中的 **内部使用区域** 有6位，编号从0开始。```

**新增机制**

当网络插件订阅到 Message\_CalibrationData 消息时，请求方会持续收到响应，报文编号为17001，json区为空，数据区为序列化的 Message\_CalibrationData 消息。
