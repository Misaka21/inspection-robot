**上传机器人文件**

**请求**

- 编号: 1799 (0x0707)
- 名称: robot\_status\_uploadfile\_req
- 描述: 上传机器人文件
- JSON 数据区: 见下表

```**字段名**|**类型**|**描述**|**可缺省**``` :- | :- | :- |
|<p>type</p><p></p>|string|机器人文件的类型，“app”=机器人程序目录下的文件，“users”=机器人用户目录下的文件|否|
|file\_path|string|机器人文件的相对路径，针对于 type 指定的文件类型|<p>否</p><p></p>|
|文件二进制数据|文件二进制数据|文件二进制数据|文件二进制数据|


```需要注意，此 API 数据区分为两个部分： **json数据区以及文件二进制数据区** 。两种数据需要紧密排列。排列顺序为：json数据+文件二进制区。在此情况下，包头中的 **数据区长度** 为json数据长度+文件二进制数据长度。包头中的 **内部使用区域** 中的第2位和第3位需要设置为json数据区的长度。```


```包头详情可见 [此处 ](..\概述\API报文结构.md)，其中的 **内部使用区域** 有6位，编号从0开始。```


```示例程序python可见此处：```
**[robot\_status\_uploadfile\_req.py]**

**请求示例**

如下表示上传文件 template 到机器人程序目录下 assets/pcd/template 处

```JSON
{
    "type": "app",
    "file\_path": "assets/pcd/template"
}```

**响应**

- 编号: 11799 (0x2E17)
- 名称: robot\_status\_uploadfile\_res
- 描述: 上传机器人文件的响应
- JSON 数据区: 见下

```**字段名**|**类型**|**描述**|**可缺省**``` :- | :- | :- |
|ret\_code|number|API 错误码|是|
|create\_on|string|API 上传时间戳|是|
|err\_msg|string|错误信息|是|
**响应示例**

```JSON
{
        "create\_on": "2022-10-12T14:06:51.035Z",
        "ret\_code": 0
}```
