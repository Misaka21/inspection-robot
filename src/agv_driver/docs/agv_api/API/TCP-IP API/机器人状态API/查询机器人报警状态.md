**查询机器人报警状态**

**请求**

- 编号: 1050 (0x041A)
- 名称: robot\_status\_alarm\_req
- 描述: 查询机器人当前的报警状态
- JSON 数据区: 无

**请求示例**


```Plaintext
5A 01 00 01 00 00 00 00 04 1A 00 00 00 00 00 00```

**响应**

- 编号: 11050 (0x2B2A)
- 名称: robot\_status\_alarm\_res
- 描述: 查询机器人当前的报警状态的响应
- JSON 数据区: 见下表

```**字段名**|**类型**|**描述**|**可缺省**``` :- | :- | :- |
|<p>fatals</p><p></p>|array[object]|<p>[报警码 Fatal](..\附录\报警码.md)</p><p>` `的数组, 所有出现的 Fatal 报警都会出现在数据中, object 格式见下文</p>|是|
|errors|array[object]|<p>[报警码 Error](..\附录\报警码.md)</p><p>` `的数组, 所有出现的 Error 报警都会出现在数据中, object 格式见下文</p>|是|
|warnings|array[object]|<p>[报警码 Warning](..\附录\报警码.md)</p><p>` `的数组, 所有出现的 Warning 报警都会出现在数据中, object 格式见下文</p>|是|
|notices|array[object]|<p>Notice</p><p>` `的数组, 所有出现的 Notice 报警都会出现在数据中, object 格式见下文</p>|是|
|ret\_code|number|API 错误码|是|
|create\_on|string|API 上传时间戳|是|
|err\_msg|string|错误信息|是|
**报警码 object 格式**

```**字段名**|**类型**|**描述**|**可缺省**``` :- | :- | :- |
|<报警码本身>|int|<报警码本身>：key 为报警码, 对应的值为自 Thu Jan 01 08:00:00 1970 至报警码产生时经过的秒数|否|
|desc|string|报警码 对应的英文描述|否|
|times|int|报警码 出现的次数|否|
|describe|string|报警码 的解决方法的描述； >3.4.6.9|否|
|method|string|报警码 具体的解决方法； >3.4.6.9|否|
|reason|string|报警码可能会出现的原因； >3.4.6.9|否|


```需注意：describe，method和reason在3.4.6.9版本之后的版本才加入```


```JSON
// key 为报警码, 对应的值为自 Thu Jan 01 08:00:00 1970 至报警码产生时经过的秒数, desc 为错误码对应的英文描述, times 为次数 
{
    "52900":1697435489,

    "describe":"third-party error",
    "method":"",
    "reason":"",
    "times":12
}```
