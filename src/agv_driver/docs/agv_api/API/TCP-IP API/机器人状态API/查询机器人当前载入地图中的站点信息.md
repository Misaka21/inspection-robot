**查询机器人当前载入地图中的站点信息**

**请求**

- 编号: 1301 (0x0515)
- 名称: robot\_status\_station\_req
- 描述: 查询机器人当前载入地图中的站点信息
- JSON 数据区: 无

**请求示例**


```Plaintext
5A 01 00 01 00 00 00 00 05 15 00 00 00 00 00 00 ```

**响应**

- 编号: 11301 (0x2C25)
- 名称: robot\_status\_station\_res
- 描述: 查询机器人当前载入地图中的站点信息的响应
- JSON 数据区: 见下表

```**字段名**|**类型**|**描述**|**可缺省**``` :- | :- | :- |
|stations|array[object]|站点数组, 若地图中没有站点, 则为空数组, object 形式见下文|是|
|ret\_code|number|API 错误码|是|
|create\_on|string|API 上传时间戳|是|
|err\_msg|string|错误信息|是|
object 形式如下:

```JSON
{
    "id": "LM1",        // 站点的 id
    "type": "LocationMark", // 站点的类型 LocationMark, ChargePoint ...
    "x": 1.23,          // 站点在世界坐标系中的 x 坐标(m)
    "y": 4.56,          // 站点在世界坐标系中的 y 坐标(m)
    "r": 1.57,          // 站点在世界坐标系中的朝向角(rad)
    "desc": "xxxxx"     // 站点的备注，如果是中文，则为 Unicode 编码, 如: \u5907\u6ce8
}```
如果附加了站点属性，则 object 形式如下：

```JSON
{
  "desc":"",
  "executor":"fork",          // 执行的操作：fork、jack、PGV、hook、DMS、None 等
  "id":"AP108",
  "prepoint":"LM100",         // 前置点      
  "r":0,
  "recfile":"p0002.pallet",   // 识别文件
  "spin":true,                // 是否启用随动
  "type":"ActionPoint",
  "use\_down\_pgv":true,        //是否使用下视PGV
  "x":-22.73,
  "y":10.135
}```
