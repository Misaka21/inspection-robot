**查询机器人文件列表**

**请求**

- 编号: 1798 (0x0706)
- 名称: robot\_status\_listfile\_req
- 描述: 查询机器人文件列表
- JSON 数据区: 见下表

```字段名|**类型**|**描述**|**可缺省**``` :- | :- | :- |
|<p>type</p><p></p>|string|机器人文件的类型，“app”=机器人程序目录下的文件，“users”=机器人用户目录下的文件|否|
|path|string|机器人文件夹的相对路径，针对于 type 指定的文件类型|否|

**请求示例**

如下表示查询机器人资源目录下 resources/rawmaps 文件夹下的目录列表

```JSON
{
        "type": "users",
        "path": "resources/rawmaps"
}```

**响应**

- 编号: 11798 (0x2E16)
- 名称: robot\_status\_listfile\_res
- 描述: 查询机器人文件列表 的响应
- JSON 数据区: 见下

```字段名|**类型**|**描述**|**可缺省**``` :- | :- | :- |
|type|string|机器人文件的类型，“app”=机器人程序目录下的文件，“users”=机器人用户目录下的文件|否|
|path|string|机器人文件夹的相对路径，针对于 type 指定的文件类型|否|
|entries|object array|为目录列表的json数组，每一个目录为一个json对象。其中包含目录名 name:string 和文件夹标识位 is\_directory:bool ，详情见示例。|是|
|ret\_code|number|API 错误码|是|
|create\_on|string|API 上传时间戳|是|
|err\_msg|string|错误信息|是|

**响应示例**


```JSON
{
  "create\_on": "2022-10-31T15:55:42.120Z",
  "entries": [{
    "is\_directory": false,
    // 是否为文件夹：false=否，true=是
    "name": "robokit\_2022-08-19\_15-45-50.rawmap" // 文件名
  },
  {
    "is\_directory": false,
    "name": "robokit\_2022-08-19\_15-45-42.rawmap"
  },
  {
    "is\_directory": false,
    "name": "robokit\_2022-08-19\_15-45-59.rawmap"
  }],
  "path": "resources/rawmaps",
  "ret\_code": 0,
  "type": "users"
}```
