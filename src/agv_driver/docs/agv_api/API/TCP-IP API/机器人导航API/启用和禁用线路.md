**启用和禁用线路**

（3.4.6.13+）

**请求**

- 编号: 3059 (0x0BF3)
- 名称: robot\_task\_path\_req
- 描述: 启用和禁用线路
- JSON 数据区: 见下表
1. 禁用线路

```字段名|类型|描述|可缺省``` :- | :- | :- |
|disablePath|json string|禁用的线路名称，json string 格式为{"id":"线路名称"}|否|
2. 启用线路

```字段名|类型|描述|可缺省``` :- | :- | :- |
|enablePath|json string|启用的线路名称，json string 格式为{"id":"线路名称"}|否|
3. 查询线路

见请求示例。

**请求示例**

1. 禁用线路

```JSON
{"disablePath":{"id":"LM1-LM2"}}```
2. 启用线路

```JSON
{"enablePath":{"id":"LM1-LM2"}}```
3. 查询线路

```JSON
["getDisablePath"]```
**响应**

- 编号: 13059 (0x3303)
- 名称: robot\_task\_path\_res
- 描述: 启用和禁用线路
- JSON 数据区: 见下表
1. 禁用线路

正常返回 "True"；如果json格式异常在err\_msg中返回解析异常的字符串，异常时如下。

```字段名|类型|描述|可缺省``` :- | :- | :- |
|ret\_code|number|API 错误码|是|
|create\_on|string|API 上传时间戳|是|
|err\_msg|string|错误信息|是|
2. 启用线路

正常返回 "True"；如果json格式异常在err\_msg中返回解析异常的字符串，异常时如下。

```字段名|类型|描述|可缺省``` :- | :- | :- |
|ret\_code|number|API 错误码|是|
|create\_on|string|API 上传时间戳|是|
|err\_msg|string|错误信息|是|
3. 查询线路

如查询前已禁用线路，则返回 被禁用线路，格式为["站点名-站点名"]；如未禁用线路，返回 "No disabled path."；如果json格式异常返回解析异常的字符串，异常时如下。

```字段名|类型|描述|可缺省``` :- | :- | :- |
|ret\_code|number|API 错误码|是|
|create\_on|string|API 上传时间戳|是|
|err\_msg|string|错误信息|是|
**响应示例**


```JSON
//禁用或启用线路正常时
True
//查询线路 已禁用线路时
["LM1-LM2"]
//查询线路 未禁用线路时
No disabled path.```
